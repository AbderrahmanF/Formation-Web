Connaissance basic sur Internet

Chaque site web est composé de 2 élément : un serveur web et un nom de domaine

Terminologie

  • Paquet : une petite unité de données transmise sur Internet.
  • Routeur : Un appareil qui dirige les paquets de données entre différents réseaux.
  • Adresse IP : identifiant unique attribué à chaque appareil sur un réseau, utilisé pour acheminer les données vers la bonne destination.
  • Nom de domaine : un nom lisible par l’homme utilisé pour identifier un site Web, tel que google.com.
  • DNS : Le système de noms de domaine est responsable de la traduction des noms de domaine en adresses IP.
  • HTTP : Le protocole de transfert hypertexte est utilisé pour transférer des données entre un client (tel qu'un navigateur Web) et un serveur (tel qu'un site Web).
  • HTTPS : une version cryptée de HTTP utilisée pour fournir une communication sécurisée entre un client et un serveur.
  • SSL/TLS : les protocoles Secure Sockets Layer et Transport Layer Security sont utilisés pour fournir une communication sécurisée sur Internet.

Comment fonctionne internet ?

How does internet work ?

Internet est un fil il permet de relier plusieurs ordinateur entre eux

Les info sur internet circule d'un ordinateur a un autre sous forme de bits sur divers supports, notamment les câbles Ethernet, les câbles a fibre optique et les signaux sans fil (onde radio)

What is HTTP ?

C’est la langue que parlent ton navigateur (Chrome, Safari, etc.) et les serveurs web pour s’envoyer des informations.

C’est grâce à HTTP que tout le web fonctionne : chaque fois que tu visites un site, ton navigateur utilise HTTP.

  1. Quand tu tapes une adresse dans ton navigateur (ex: www.google.com), il envoie une requête HTTP au serveur.
  2. Le serveur web (un ordinateur spécialisé qui héberge le site) reçoit ta demande. Il vérifie si la page existe, puis il prépare une réponse HTTP.
  3. Le serveur te répond avec : de statut (ex : 200 = succès, 404 = page introuvable) Le contenu demandé (HTML, image, JSON, etc.)

Méthode HTTP :

  • Get : obtient des données (ex: page web)
  • Post : Envoyer des données
  • Put : Mettre a jour une ressource
  • Delete : supprimer une ressource

What is hosting : hébergement Web

def: Pour qu'un site web soit fonctionnel deux étape sont nécessaire : Enregistrer un nom de domaine et souscrire à un hébergement, ces deux éléments sont essentiels et votre site web ne pourra pas exister sans eux

Lorsque votre site web est uniquement héberger sur votre ordinateur les autres n'y ont pas accès.

l'hébergement web est le service qui vous permet de louer un espace sur internet pour le contenue de votre site pour le rendre accesible au monde entier

Principaux type d'hébergement web :

  • hébergement mutualisé
  • hébergement dédiée
  • serveur privé virtuel (VPS)
  • hébergement revendeur
  • Hébergement cloud

Comment fonctionne l'hébergement Web

une fois le le nom de domaine enregistrer les fichiers qui contiennent le contenu de votre site web(HTML, CSS, etc..) doivent être stockée ensemble dans un emplacement connecté à internet : Un serveur web

L'hébergement web consiste à télécharger les fichiers d'un site web depuis un ordinateur local vers un serveur web, puis à les publier en ligne. Les ressources du serveur, notamment le processeur, la RAM, l'espace disque et la bande passante, sont allouées aux sites web qu'il héberge

Hébergement Web partagé

Lorsqu'un seul serveur physique héberge plusieurs sites web, on parle d'hébergement mutualisé

Hébergement VPS

est un type d'hébergement où un serveur physique est divisé par un logiciel de virtualisation et attribué à différents clients. Ainsi, chaque utilisateur dispose d'un serveur virtuel isolé des autres

Hébergement dédié

est l'option la plus puissante et la plus efficace. Un serveur physique complet et toutes ses ressources sont alloués à un seul client

Hébergement cloud

Il s'agit du dernier type d'hébergement sur le marché, qui a connu un succès considérable ces dernières années. La technologie cloud consiste à répartir les données sur plusieurs serveurs virtuels qui forment le cloud. L'hébergement cloud rend toute application web ou site web accessible via un réseau cloud, offrant ainsi une flexibilité, des performances et une rapidité accruesx²

Hébergement revendeur

L'hébergement web où le propriétaire du compte peut mettre son espace disque dur et sa bande passante à disposition pour héberger des sites web tiers est appelé hébergement revendeur . Le propriétaire initial du compte d'hébergement est le « revendeur ».

DNS

What is DNS ?

Le système DNS est un annuaire téléphonique pour internet, les utilisateurs accèdent aux information en ligne via des Nom de domaine

Les navigateurs interagissent via des add IP.

Le dns traduit c'est Nom de domaine en IP afin que les navigateurs puissent accèder aux ressource d'internet.

Le processus de résolution DNS consiste à convertir un nom d'hôte en une adresse IP conviviale

Comment fonctionne le DNS ?

Le processus de résolution DNS consiste à convertir un nom d'hôte en une IP

Lorsqu'un utilisateur souhaite consulter une page web, une conversion doit être effectuée entre ce qu'il saisit dans son navigateur et l'adresse conviviale nécessaire pour localiser la page web

4 serveur DNS impliqué dans le chargement d'une page web

  • Récurseur DNS
chargé de rechercher un livre spécifique dans une bibliothèque. Le récurseur DNS est un serveur conçu pour recevoir les requêtes des ordinateurs clients via des applications telles que des navigateurs web

  • serveur racine
est la première étape de la traduction (résolution) des noms d'hôtes lisibles par l'homme en adresses IP. Il peut être comparé à l'index d'une bibliothèque pointant vers différents rayonnages de livres ; il sert généralement de référence à des emplacements plus spécifiques

  • Serveur de noms TLD
Le serveur de domaine de premier niveau ( TLD ) peut être comparé à un rayonnage de livres dans une bibliothèque. Ce serveur de noms constitue l'étape suivante dans la recherche d'une adresse IP spécifique et héberge la dernière partie d'un nom d'hôte (dans example.com, le serveur TLD est « com »).

  • Serveur de noms faisant autorité
ce serveur de noms final peut être comparé à un dictionnaire sur une étagère, dans lequel un nom spécifique peut être traduit en sa définition. Le serveur de noms faisant autorité est le dernier point de la requête. Si le serveur de noms faisant autorité a accès à l'enregistrement demandé, il renvoie l'adresse IP du nom d'hôte demandé au récurseur DNS (le bibliothécaire) ayant effectué la requête initiale.

Navigateur Web

tout le monde doit utiliser un navigateur web pour accéeder à internet et y établir leur présence

Le dev Web repose entièrement sur c'est le concept de navigateur web

Qu'est ce qu'un navigateur Web?

Un navigateur Web est un logiciel qui permet à un utilisateur d’accéder à des informations sur Internet via le World Wide Web.

Composant :

Moteur de rendu - L'architecture Web

Il s'agit du composant principal d'un navigateur web qui convertit le code du site web en éléments visuels visibles sur une page web Le moteur de rendu reçoit les instructions du navigateur, les interprète et crée une page web en conséquence

  • Blink : ce moteur alimente google chrome
  • WebKit: utilisée dans Safari
  • Gecko : utilisée par mozilla firefox

Interface utilisateur (IU) :

Vous interagissez directement avec elle. Elle comprend la barre d'adresse, où vous saisissez les adresses des sites web etc...

Moteur de rendu:

l'architecte est responsable de la construction de la représentation visuelle de la page Web

Composant réseau :

il récupère les fichiers du site Web (code, images, vidéos) à partir des serveurs Web du monde entier, garantissant que tous les éléments nécessaires sont livrés au moteur de rendu pour créer la page Web

Moteur JavaScript :

il interprète et exécute le code JavaScript, permettant aux pages Web de répondre aux actions des utilisateurs et de créer des expériences dynamiques .

Composants de sécurité

ils gèrent des tâches telles que le cryptage des transmissions de données (HTTPS) et vous protègent contre les sites Web malveillants.
