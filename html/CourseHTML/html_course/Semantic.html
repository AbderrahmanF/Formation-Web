Règle de semantic

pour les titre <h1> - <h6> ils doivent se suivre les un aux autres c'est a dire qu'apres un h1 on doit avoir un h2 et pas un h3 ainsi de suite mais que dans un conteneur on peut mettre h2 apres h3 puis après repartir sur du h2 dans un autres conteneur.

hiérarchie structurelle

Utiliser une hiérarchie appropriée est essentiel pour l'accessibilité. Les technologies d'assistance, comme les lecteurs d'écran, s'appuient sur la structure d'une page web pour déterminer comment l'analyser et l'annoncer à l'utilisateur. L'utilisation d'un h3élément après un élément h1peut donner à penser à l'utilisateur d'un lecteur d'écran qu'il a accidentellement ignoré un contenu important, en raison de l'absence d'un h2élément.

Une structure adéquate est également importante pour le référencement. Les moteurs de recherche utilisent l'automatisation pour analyser le contenu de votre page web et déterminer quand et où elle doit apparaître dans les résultats. Si votre structure est mal structurée, les moteurs de recherche risquent de ne pas vous positionner correctement dans les résultats de recherche pertinents.

What are Semantic Element?

A semantic element clearly describes its meaning to both the browser and the developer.

Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.

Examples of semantic elements: <img>, <table>, and <article> - Clearly defines its content.

In HTML there are several semantic elements that can be used to define different parts of a web page:

HTML <section> element

The <section> element defines a section in a document.

A section is a thematic grouping of content, typically with a heading.

HTML <article> Element

The <article> element specifies independent, self-contained content.

An article should make sense on its own, and it should be possible to distribute it independently from the of the web site.

HTML <header> Element

The <header> element represents a container for introductory content or a set of navigational links.

HTML <footer> Element

The <footer> element defines a footer for a document or section.

HTML <nav> Element

The <nav> element defines a set of navigation links.

HTML <aside>

The <aside> element defines some content aside from the content it is placed in (like a sidebar).

The <aside> content should be indirectly related to the surrounding content.

HTML <figure> and <figcaption> Elements

The <figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.

The <figcaption> tag defines a caption for a <figure> element. The <figcaption> element can be placed as the first or as the last child of a <figure> element.
