Rappel des notion global sur l'accessibilité

Introduction à l'accessibilité

Règles pour l'accessibilité des contenus Web Les Règles pour l'accessibilité des contenus Web (WCAG) constituent un ensemble de directives visant à rendre les contenus Web plus accessibles aux personnes handicapées. Les quatre principes des WCAG sont : POUR (Perceptible, Exploitable, Compréhensible et Robuste)

Technologie d'assistance pour l'accessibilité

  • Lecteurs d'écran : logiciels qui lisent à voix haute le contenu d'un écran d'ordinateur. Ils sont utilisés par les personnes aveugles ou malvoyantes pour accéder au web.
  • Claviers à grand texte ou en braille : utilisés par les personnes malvoyantes pour accéder au Web.
  • Loupes d'écran : Logiciels permettant d'agrandir le contenu d'un écran d'ordinateur. Elles sont utilisées par les personnes malvoyantes pour accéder au web.
  • Dispositifs de pointage alternatifs : utilisés par les personnes à mobilité réduite pour accéder au Web. Cela inclut des dispositifs tels que les joysticks, les trackballs et les pavés tactiles.
  • Reconnaissance vocale : Utilisée par les personnes à mobilité réduite pour accéder au web, elle permet aux utilisateurs de contrôler un ordinateur à la voix.

Outils d'audit d'accessibilité

Outils d'accessibilité courants : Google Lighthouse, Wave, IBM Equal Accessibility Checker et axe DevTools sont quelques-uns des outils d'accessibilité courants utilisés pour auditer l'accessibilité d'un site Web.

Meilleures pratiques en matière d'accessibilité

  • Structure de titres appropriée : utilisez des niveaux de titres appropriés pour créer une structure logique pour votre contenu. Cela permet aux utilisateurs de technologies d'assistance de mieux comprendre le contenu de votre site web.
  • Accessibilité et tableaux : Lorsque vous utilisez des tableaux, utilisez l' th élément pour définir les cellules d'en-tête et l' td élément pour définir les cellules de données. Cela permet aux utilisateurs de technologies d'assistance de comprendre la structure du tableau. Cet caption élément permet d'écrire la légende (ou le titre) d'un tableau, afin que les utilisateurs, notamment ceux qui utilisent des technologies d'assistance, puissent rapidement comprendre l'objectif et le contenu du tableau
  • Importance d'associer une étiquette aux entrées : et élément permet label d'associer des étiquettes aux entrées du formulaire. Cela permet aux utilisateurs de technologies d'assistance de comprendre l'objectif de l'entrée.
  • Importance d' alt un texte de qualité : utilisez cet alt attribut pour fournir une alternative textuelle aux images. Cela permet aux utilisateurs de technologies d'assistance de comprendre le contenu de l'image.
  • Importance d'un texte de lien de qualité : utilisez un texte descriptif pour aider les utilisateurs à comprendre l'objectif du lien. Cela permet aux utilisateurs de technologies d'assistance de comprendre l'objectif du lien.
  • Bonnes pratiques pour rendre les contenus audio et vidéo accessibles : Fournissez des sous-titres et des transcriptions pour les contenus audio et vidéo afin de les rendre accessibles aux personnes malentendantes. Fournissez également des descriptions audio pour les contenus vidéo afin de les rendre accessibles aux personnes malvoyantes.
  • tabindex Attribut : permet de rendre les éléments accessibles et de définir l'ordre de navigation au clavier. Il est important de ne jamais utiliser cet tabindex attribut avec une valeur supérieure à 0.
  • accesskey Attribut : permet de définir un raccourci clavier pour un élément. Cela peut faciliter la navigation sur le site web pour les utilisateurs à mobilité réduite.

WAI-ARIA, rôles et attributs

  • WAI-ARIA : Web Accessibility Initiative - Accessible Rich Internet Applications (WAI-ARIA) est un ensemble d'attributs pouvant être ajoutés aux éléments HTML pour améliorer l'accessibilité. Il fournit des informations complémentaires aux technologies d'assistance sur l'objectif et la structure du contenu.
  • Rôles ARIA : ensemble de rôles prédéfinis pouvant être ajoutés aux éléments HTML pour en définir l'objectif. Cela permet aux utilisateurs de technologies d'assistance de comprendre le contenu du site web. Exemples : role="tab" , role="menu" et role="alert".

aria-label et aria-labelledby attributs : ces attributs permettent de donner à un élément un nom programmatique (ou accessible), ce qui aide les utilisateurs de technologies d'assistance (comme les lecteurs d'écran) à comprendre son objectif. Ils sont souvent utilisés lorsque l'étiquette visuelle d'un élément est une image ou un symbole plutôt qu'un texte. aria-label Ils permettent de définir le nom directement dans l'attribut et aria-labelledby de référencer du texte existant sur la page.

aria-hidden : permet de masquer un élément aux technologies d'assistance telles que les lecteurs d'écran. Par exemple, cela peut servir à masquer des images décoratives qui n'apportent aucun contenu significatif.

aria-expanded : utilisé pour transmettre l'état d'une fonction de bascule (ou de divulgation) aux utilisateurs de lecteurs d'écran.

aria-live : utilisé pour indiquer que le contenu d'un élément est suffisamment important pour nécessiter que toute modification soit immédiatement annoncée aux utilisateurs de lecteurs d'écran

États ARIA courants : les états ARIA courants incluent

  • aria-haspopup
  • aria-checked
  • aria-disabled
  • aria-selected
Ces attributs peuvent être utilisés pour indiquer l'état d'un élément et aider les personnes utilisant des technologies d'assistance à comprendre le contenu du site Web.

aria-haspopup : cet état indique qu'un élément interactif déclenchera une fenêtre contextuelle lorsqu'il sera activé. Il ne peut être utilisé que si la fenêtre contextuelle possède l'un des rôles suivants :

  • menu
  • listbox
  • tree
  • grid
  • dialog
La valeur de aria-haspopup doit être l'un de ces rôles ou true, qui est identique à menu

aria-checked : cet attribut permet d'indiquer si un élément est coché. Il est généralement utilisé lors de la création de cases à cocher, de boutons radio, de commutateurs et de listes déroulantes personnalisés.

aria-disabled : Cet état est utilisé pour indiquer qu'un élément est désactivé uniquement pour les personnes utilisant des technologies d'assistance, telles que les lecteurs d'écran.

aria-selected : cet état indique qu'un élément est sélectionné. Vous pouvez l'utiliser sur des contrôles personnalisés comme une interface à onglets, une liste déroulante ou une grille.

aria-controls : permet d'associer un élément à un autre élément qu'il contrôle. Cela permet aux utilisateurs de technologies d'assistance de comprendre la relation entre les éléments.

aria-describedby Utilisé pour fournir des informations supplémentaires sur un élément en l'associant à un autre élément contenant ces informations. Cela permet aux utilisateurs de lecteurs d'écran d'accéder immédiatement aux informations supplémentaires lorsqu'ils naviguent vers l'élément. L'utilisation courante consiste à associer des instructions de formatage à une saisie de texte ou un message d'erreur à une saisie après une soumission de formulaire non valide.
